INSTRUCTIONS
────────────
For each task below, assign ONE tier:

  A — AI Assist (full confidence)
      AI output can be used directly with minimal review.
      No production action is taken based on the output.

  B — AI Assist (validate first)
      AI produces a draft or recommendation.
      An engineer MUST verify and approve before acting.

  C — Automation / scripting only
      Deterministic, repeatable task. AI adds no value —
      a script is more reliable and consistent.

  D — Human decision required
      Risk, compliance, judgment, or irreversibility involved.
      AI must NOT make or action this decision.

Work individually for 20 minutes, then compare with your tower.
Record your Tier and a one-sentence reason for each task.

Reason:  Will help in RCA and justification

is running, and if the health check endpoint returns a
non-200 response, restarts the service automatically.

Tower domain:  Compute
Your tier: B 
Reason: Engineer should validate

TASK 4
A change window request asks to add an NSG inbound rule
allowing port 443 from source 0.0.0.0/0 on the production
payment gateway subnet. Approve or reject this change.

Tower domain:  Network
Your tier:  D
Reason:Involves production network security architecture compliance and critical risk mitigation

TASK 5
Analyse FinBridge's PostgreSQL slow query log from the last
24 hours. Identify which queries are missing indexes and
suggest the specific index commands to create.

Tower domain:  DB
Your tier: B
Reason:he AI can analyze logs and draft index optimization suggestions, but a database engineer must validate them to prevent potential performance degradation or indexing overhead.

TASK 6
Recommend the correct Azure VM SKU and disk type for a new
FinBridge batch processing workload. Requirements: 4 vCPU,
16GB RAM peak, 500 IOPS sustained, cost-optimised.

Tower domain:  Compute
Your tier: B
Reason:  The AI can analyze hardware requirements to recommend cost-optimized Azure configurations, but an engineer must verify the choice against real-world performance workloads and budget compliance before deployment.

TASK 7
Delete all log files older than 30 days from /var/log/app/
on the FinBridge application server. Run daily at 02:00 UTC.

Tower domain:  Compute
Your tier: C
Reason:  Cleaning up files older than 30 days is a deterministic, repeatable maintenance task that is safely and reliably handled by a standard system script (like a cron job) rather than an AI.

TASK 8
The FinBridge primary production database is unresponsive
during a major incident. Determine whether to fail over to
the secondary replica immediately or wait for recovery.

Tower domain:  DB
Your tier:  D
Reason: Production failover during a major incident carries high business risk, financial impact, and irreversible consequences, requiring critical human judgment and compliance oversight.

TASK 9
Generate Terraform code for a new Azure VNet (10.0.0.0/16)
with one app subnet (10.0.1.0/24), one db subnet (10.0.2.0/24),
and an NSG that allows SSH only from the Bastion subnet.

Tower domain:  Network
Your tier:  B
Reason:  * Code requires human validation.
* Misconfigurations risk infrastructure security breaches.
* Human oversight prevents deployment errors.

TASK 10
Review the current NSG ruleset for the FinBridge payment
processing subnet. Identify any rules that introduce
security risk and explain why each is a concern.

Tower domain:  Network
Your tier:  B
Reason:  * Security reviews demand expert oversight.
* AI can miss complex vulnerabilities.
* Human verification prevents critical oversights.

TASK 11
The nightly backup job for the final data centre copy is
causing severe I/O contention, affecting production payment
writes. Decide whether to cancel the backup job or continue.

Tower domain:  Storage
Your tier: D
Reason: * Decisions involve high financial risks.
* Data integrity requires human accountability.
* Business continuity demands critical judgment.

TASK 12
Using 90 days of CPU and memory utilisation metrics for
vm-app, forecast when the current Standard_B2ms VM will
need to be resized, and recommend the target SKU.

Tower domain:  Compute
Your tier:  B
Reason: * Financial impacts require human validation.
* Forecasting requires capacity planning expertise.
* Human review prevents expensive over-provisioning.

TASK 13
Monitor all FinBridge storage volumes and send an automated
alert to the operations channel when any volume exceeds
80% utilisation.

Tower domain:  Storage
Your tier:  C
Reason:  This is a purely deterministic, threshold-based monitoring task handled reliably by standard alerting scripts.

TASK 14
Analyse access frequency logs for 50 FinBridge storage
volumes over 90 days. Propose which volumes should be
migrated from hot tier to cool tier to reduce costs.

Tower domain:  Storage
Your tier: B
Reason: AI can look for trends in data to recommend cost-saving optimization drafts, but an engineer must verify them before moving infrastructure.

TASK 15
Using the engineering RCA and incident timeline, write a

Tower domain:  All towers
Your tier: B
Reason: Generative AI excel at digesting technical notes into a draft summary, though human review is required to guarantee factual alignment.

TASK 16
FinBridge Finance is evaluating a 3-year Azure Reserved
Instance commitment for 30 production VMs — total cost
£240,000 upfront. Approve or reject this commitment.

Tower domain:  Cloud / Finance
Your tier: D
Reason:  This involves high financial risk, legal/compliance commitments, and strategic business judgment that a human must decide.

TASK 17
A schema migration added an index on orders.customer_id
and is causing connection pool exhaustion. Generate the
rollback SQL script to drop the index safely.

Tower domain:  DB
Your tier: B
Reason: AI can accurately draft the SQL script to drop the index. However, executing database commands in production carries high risk. An engineer must verify and approve the script before execution.

TASK 18
A new storage replication vendor is proposing an SLA of
99.5% availability for FinBridge's DR replication service.
Decide whether this SLA is acceptable for FinBridge's needs.

Tower domain:  Storage
Your tier: D
Reason:  Determining SLA acceptability requires business judgment, risk assessment, and compliance alignment. AI must not make business-critical decisions regarding disaster recovery standards.

TASK 19
Analyse 200 FinBridge service accounts. Using last-login
date and role data, produce a list of accounts that are
candidates for deactivation.

Tower domain:  Compute / Security
Your tier: C
Reason: Parsing login dates and role data is completely rule-based and deterministic. A standard code script is more reliable, accurate, and consistent than an AI model.

TASK 20
Using the provided log files and timeline from the Friday
night network outage, produce the post-incident 5-Why
analysis document.

Tower domain:  Network
Your tier: B
Reason:AI excels at synthesizing text from log timelines into a structured 5-Why format. However, a human engineer must validate the technical accuracy of the root-cause analysis before finalizing the report.
